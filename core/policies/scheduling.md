# Scheduling Policy

## Canonical behavior

1. Always check the live calendar before proposing times.
2. Propose exact day/date/time values, not vague windows.
3. Propose exactly 2 slots by default unless the user requests a different count.
4. Respect travel buffers for in-person meetings.
5. If no valid slots exist, state that clearly and offer the nearest alternatives.
